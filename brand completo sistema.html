01 — Fundamentos

Filosofia da Marca

Cinnamon é a evolução do operacional financeiro tradicional. Uma plataforma que transforma processos manuais em fluxos inteligentes, mantendo a precisão institucional que o mercado financeiro exige.

P

Precisão Institucional

Cada elemento visual comunica confiabilidade. Sem excessos, sem ruído. Apenas a informação necessária, apresentada com clareza absoluta.

S

Sofisticação Acessível

Entre big four e fintech disruptiva. Tecnologia enterprise sem burocracia. Interfaces complexas que parecem simples.

E

Eficiência Visual

Design que reduz fadiga operacional. Hierarquia clara, contraste intencional, densidade controlada. Produtividade como resultado do design.

"Construída de dentro para fora. Nasceu da operação real de um FIDC, não de hipóteses de consultoria."

— Manifesto Cinnamon

Posicionamento Visual

Operações que funcionam. Fundos que crescem. — Esta é a essência da marca. O sistema visual reflete exatamente isso: estrutura que funciona, evolução que não quebra, crescimento sustentável.

A paleta cromática âncora-se no bordô institucional (Cinnamon 700), expandindo para neutros premium e superfícies quentes. Evitamos azuis corporativos saturados e verdes fintechs. Nosso território é o calor controlado, a sofisticação discreta, a autoridade sem arrogância.

05 — Diretrizes UX

UX para Plataformas Financeiras

Princípios de experiência construídos para ambientes de alta densidade informacional. Reduzir fadiga, aumentar produtividade, manter precisão.

Hierarquia Visual

  • Informação crítica sempre no topo: valores, status, alertas
  • Dados financeiros em destaque tipográfico e cromático
  • Metadados secundários em Slate 500, tamanho 13-14px
  • Separação clara entre seções com espaçamento generoso
  • Evitar mais de 3 níveis de hierarquia visual simultâneos

Densidade Controlada

  • Tabelas: máximo 10 linhas visíveis sem scroll
  • Cards KPI: 4-6 por linha em desktop, nunca mais
  • Espaçamento entre elementos: mínimo 16px (space-4)
  • Margem interna de cards: 24px (space-6) mínimo
  • Evitar excesso de borders — usar espaçamento como separador

Leitura Escaneável

  • Parágrafos: máximo 80 caracteres por linha
  • Dados numéricos sempre alinhados à direita
  • Monoespaçados obrigatórios para valores monetários
  • Labels em uppercase + letter-spacing para facilitar scan
  • Cores de status dessaturadas para não causar alarme falso

Redução de Fadiga

  • Fundo Ivory 50 reduz brilho comparado a branco puro
  • Contraste texto/background entre 7:1 e 12:1 (sweet spot 9:1)
  • Animações: 200ms máximo, ease-in-out sempre
  • Hover states sutis — sem transformações exageradas
  • Evitar vermelho saturado — usar Error 700 dessaturado

Feedback Operacional

  • Estados de loading sempre visíveis e discretos
  • Confirmações de ação em toast, não modais bloqueantes
  • Erros críticos em modal, warnings em inline alert
  • Ações destrutivas sempre com confirmação dupla
  • Sucesso confirmado com badge verde + microtransição

Navegação Eficiente

  • Sidebar fixo com seções principais sempre visível
  • Breadcrumbs em operações com múltiplos níveis
  • Atalhos de teclado documentados e consistentes
  • Filtros sempre no topo, nunca ocultos em dropdown
  • Paginação com jump to page, não apenas next/prev

Microinterações

Toda interação possui resposta visual imediata (120ms máximo de delay percebido). Hover adiciona sombra ou transforma -1px. Active adiciona inner shadow. Disabled reduz opacity para 0.5 e remove cursor pointer. Loading states usam skeleton screens em Slate 100, nunca spinners isolados.

Tratamento de Erros

Erro de campo: border Error 700 + mensagem inline abaixo do input.
Erro de página: card centralizado com ícone, mensagem clara, ação de recuperação.
Erro crítico: modal bloqueante, fundo obsidian com opacity 0.8, mensagem detalhada.
Sempre fornecer: (1) O que aconteceu, (2) Por que aconteceu, (3) O que fazer agora.

Estratégia de Dark Theme

Quando usar:

  • — Operação noturna prolongada
  • — Ambientes com pouca luz
  • — Monitoramento contínuo de dados
  • — Preferência do usuário

Vantagens UX:

  • — Redução de fadiga visual em 40%
  • — Menor emissão de luz azul
  • — Contraste otimizado para leitura
  • — Foco em dados críticos

Regras de Acessibilidade Dark:

Contraste mínimo 7:1 entre texto principal e background. Titanium metálico para accents discretos. Evitar branco puro (#FFF) — usar Ivory 100. Sombras sutis com opacity reduzida. Status colors mantêm dessaturação. Obsidian 950 como base, 900 para elevação, 800 para hover states.

Referências Conceituais

Direções visuais que aspiramos e armadilhas que evitamos. Construindo identidade entre sofisticação enterprise e modernidade fintech.

O Que Aspirar

  • Plataformas financeiras institucionais com visual contemporâneo
  • Dashboards Bloomberg Terminal — densidade informacional organizada
  • Stripe Dashboard — hierarquia clara, microinterações precisas
  • Linear — velocidade operacional, atalhos, eficiência
  • Sistemas bancários europeus — sofisticação discreta
  • Notion — organização visual sem perder simplicidade
  • Figma — componentes consistentes, estado sempre claro
  • Grafana — visualização de dados complexos legível

O Que Evitar

  • Aparência genérica de template admin bootstrap
  • Excesso de gradientes e efeitos glassmorphism
  • Cores saturadas competindo por atenção
  • Animações exageradas que atrasam operação
  • Tabelas sem hierarquia visual clara
  • Dashboards poluídos com 20+ KPIs simultâneos
  • Visual "cripto startup" com neon e excesso de glow
  • Tipografia display onde deveria haver dados mono

Território Visual

  • Entre big four (PWC, Deloitte) e fintech disruptiva
  • Sofisticação sem arrogância corporativa antiga
  • Modernidade sem exagero tecnológico
  • Precisão sem frieza de sistemas legados
  • Calor cromático controlado — bordô, titanium, ivory
  • Evitar azuis corporativos saturados
  • Evitar verdes fintech chamativos
  • Ocupar espaço entre autoridade e acessibilidade

Sensação Alvo

  • "Finalmente um sistema que pensa como eu opero"
  • "Sofisticado mas não me faz perder tempo"
  • "Parece caro, mas é intuitivo"

Especificações para Desenvolvimento

Diretrizes técnicas para implementação consistente em Django templates. Grid, espaçamentos, componentes, comportamentos.

Mobile < 768px

Grid System

Columns 12 (desktop)

Gap Compacto 16px (space-4)
Auto-fit Min 280px (cards)
KPI Grid 200px minmax

Django Templates

Classes CSS devem seguir nomenclatura BEM quando necessário: .card, .card__header, .card__title. Componentes reutilizáveis em partials/. Variáveis CSS (custom properties) sempre prefixadas com --. Cores nunca hardcoded — sempre via variável. Espaçamentos sempre múltiplos de 4px.

Acessibilidade

Contraste mínimo: 7:1 para texto normal, 4.5:1 para texto grande.
Focus visible: Sempre presente, outline ou shadow ring.
Keyboard navigation: Tab order lógico, skip links em dashboards.
ARIA labels: Obrigatório em ícones sem texto, gráficos, status.
Color blindness: Status nunca apenas por cor — adicionar ícone ou texto.
